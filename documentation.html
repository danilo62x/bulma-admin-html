    Introdução

    Bem-vindo à documentação do Admin Template. Este guia cobre desde a instalação até o uso avançado dos componentes. O template é construído com HTML, Bulma e JavaScript vanilla, priorizando performance e suporte completo a modo escuro.

    Dica

    Você pode servir o template com qualquer servidor estático — não há necessidade de build pesado.

    Instalação

    Clone o repositório e sirva os arquivos estáticos:

    # Clonar o projeto
git clone https://github.com/acme/admin-template.git
cd admin-template/html

# Servir localmente
npx serve .

    A aplicação ficará disponível em http://localhost:3000.

    Configuração

    As cores e tokens de design são definidos em assets/css/variables.css:

    :root {
  --tx-primary: #485fc7;
  --tx-success: #48c774;
  --tx-radius: 6px;
}

    Atenção

    O CSS usa nesting nativo. Para navegadores antigos, compile com PostCSS.

    Componentes

    O template inclui um conjunto de componentes reutilizáveis. A tabela abaixo resume os principais:

    ComponenteDescriçãoUso
    tx-cardContainer de seção com cabeçalho.header, body
    tx-stats-cardCartão de métrica com tendência.icon, value, label
    tableTabela Bulma com sort/filter via JS.is-fullwidth, is-hoverable
    modalDiálogo modal com toggle por classe.is-active

    API

    A API REST utiliza autenticação via token Bearer. Inclua o cabeçalho Authorization em todas as requisições autenticadas:

    curl https://api.acme.com.br/v1/users \
  -H "Authorization: Bearer SEU_TOKEN" \
  -H "Content-Type: application/json"
    MétodoEndpointDescrição
    GET/v1/usersLista todos os usuários.
    POST/v1/usersCria um novo usuário.
    DELETE/v1/users/:idRemove um usuário.
